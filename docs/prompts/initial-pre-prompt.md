Necesito crear un sistema que genere una página estática con subpáginas para productos a vender. La idea es simplificar la publicación de una venta de garage, ya que voy a vender cosas que no necesito para una mudanza. Sin base de datos o algo muy simple (estoy abierto a usar markdown para cada elemento publicado)

Tiene que permitirme: Cargar productos con una descripción, título, múltiples imágenes de las cuales una es la principal o imagen de publicación, y posiblemente videos, incluir precio, etiquetas, posiblemente datos estructurados en tablas como medidas, bullet points, etc.

Cada elemento a vender tendrá un estatus de normal/reservado/vendido (los elementos vendidos estarán disponibles para ver pero no podrán ser reservados o comprados.

Los posibles compradores podrán:
- ver la lista de cosas a vender
- filtrarla/ordenarla (normalmente se mostrarán los no vendidos primero.
- contactar al vendedor para verlos/reservarlos

El vendedor (en esta versión 1, solo yo, los cambios puedo hacerlos como commits para evitarme armar un backend complejo y autenticación), podrá: 
- cargar y editar productos, fotos y videos, títulos, descripciones y precios
- actualizar el status de los productos, eliminarlos definitivamente u ocultarlos
- de una forma sencilla (clic para copiar cada elemento) cross-post en otros lugares (por ejemplo, mercadolibre, facebook marketplace) y en tal caso, podrá haber un link a la publicación (y un link cruzado en la publicación que haga en esos marketplaces)

- Cada publicación en la app tiene que tener una URL corta y fácil de memorizar o leer.
